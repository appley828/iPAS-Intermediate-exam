① 激活函數全解 您特別標記：sigmoid / ReLU
四大激活函數公式 & 特性
激活函數 // ① Sigmoid：把任意值壓縮到 (0,1)，常用於二元分類輸出層
σ(x) = 1 / (1 + exp(-x))

// ② Tanh：壓縮到 (-1,1)，比 Sigmoid 更零中心化
tanh(x) = (exp(x) - exp(-x)) / (exp(x) + exp(-x))

// ③ ReLU：負值歸零，正值不變（最常用隱藏層激活函數）
ReLU(x) = max(0, x)

// ④ Softmax：多類別機率輸出，Σ = 1（用於輸出層）
σ(zᵢ) = exp(zᵢ) / Σ exp(zⱼ)
激活函數比較表 ── 考試必背
激活函數輸出範圍梯度消失主要用途缺點
Sigmoid(0, 1)嚴重（兩端飽和）二元分類輸出層梯度消失、非零中心
Tanh(-1, 1)存在（兩端飽和）RNN 隱藏層梯度消失（雖比 Sigmoid 弱）
ReLU ⭐[0, +∞)無（正區間）CNN / DNN 隱藏層Dying ReLU（負值永遠為0）
Leaky ReLU(-∞, +∞)幾乎無解決 Dying ReLU需調整 α 參數
Softmax(0,1)，和=1存在多類別輸出層只適合輸出層
⚠️ 梯度消失（Vanishing Gradient）：Sigmoid / Tanh 在 |x| 很大時，導數趨近於 0，深層網路反向傳播後梯度幾乎消失 → 深層網路首選 ReLU！
✅ ReLU 為何能解決梯度消失？f'(x) = 1（當 x>0），梯度不衰減，深層網路信號可順利傳遞。

Dropout — 科三 Q39 答案 # ✅ 識別關鍵字：nn.Dropout / p=0.5 / 訓練時隨機關閉神經元 import torch.nn as nn model = nn.Sequential( nn.Linear(784, 256), nn.ReLU(), nn.Dropout(p=0.5), # ← Dropout！p=50% 的神經元被關閉 nn.Linear(256, 128), nn.ReLU(), nn.Dropout(p=0.3), # ← 不同層可設不同丟棄率 nn.Linear(128, 10) ) # 注意：訓練模式才會 Dropout，評估時自動關閉 model.train() # 開啟 Dropout model.eval() # 關閉 Dropout（推論時）
✅ 識別 Dropout：看到 nn.Dropout(p=...) 就是正則化！p=丟棄率（通常 0.2~0.5）
💡 Dropout 原理：訓練時隨機將 p 比例的神經元輸出設為 0，迫使網路學習冗餘表示，防止過擬合。
L1 / L2 正則化 # L2 正則化（PyTorch 中透過 weight_decay 設定） optimizer = torch.optim.Adam( model.parameters(), lr=0.001, weight_decay=1e-4 # ← L2 正則化！ ) # L1 正則化（需手動加入損失） l1_lambda = 0.001 l1_norm = sum(p.abs().sum() for p in model.parameters()) loss = criterion(outputs, labels) loss = loss + l1_lambda * l1_norm # ← L1 加入損失 # Sklearn LASSO（L1） from sklearn.linear_model import Lasso, Ridge lasso = Lasso(alpha=0.1) # L1 ridge = Ridge(alpha=1.0) # L2
識別特徵代表正則化
weight_decay=...L2（Ridge）
p.abs().sum() 加入 lossL1（Lasso）
Lasso(alpha=...)L1
Ridge(alpha=...)L2
Batch Normalization # ✅ 識別關鍵字：nn.BatchNorm / 在激活函數之前（或之後） model = nn.Sequential( nn.Linear(784, 256), nn.BatchNorm1d(256), # ← Batch Norm！（全連接層後） nn.ReLU(), nn.Linear(256, 10) ) # CNN 中用 BatchNorm2d nn.BatchNorm2d(64) # ← CNN 特徵圖的 BN
⚠️ Batch Norm ≠ Dropout：BN 正規化每批次特徵分佈（加速訓練），Dropout 隨機丟棄神經元（防過擬合），兩者通常同時使用但目的不同！

③ 優化器進化史 科三 Q17：哪個內建動量？
SGD → Momentum → RMSProp → Adam（逐步進化）
優化器公式 // SGD（基礎版）
w ← w - lr × ∇L

// SGD + Momentum（加入速度項 v）
v ← β × v + (1-β) × ∇L // 梯度移動平均（一階動量）
w ← w - lr × v

// Adam = Momentum（一階）+ 自適應學習率（二階）
m ← β₁ × m + (1-β₁) × ∇L // 一階動量
v ← β₂ × v + (1-β₂) × ∇L² // 二階動量
w ← w - lr × m̂ / (√v̂ + ε)
優化器內建動量？自適應學習率？適用場景
SGD❌ 無❌ 固定簡單任務、需要精確調參
SGD + Momentum⚠️ 需手動設❌ 固定加速 SGD
RMSProp❌ 無一階動量✅ 有二階RNN、非穩定梯度
Adam ⭐✅ 內建一階動量✅ 有二階最常用！幾乎所有場景
Adagrad❌ 無✅ 累積梯度平方稀疏特徵（NLP）
✅ 科三 Q17 答案 B：Adam 同時內建「一階動量（追蹤梯度移動平均）」+ 「二階動量（自適應學習率）」
❌ SGD+Momentum 雖有動量，但需要手動設置 momentum=0.9，且沒有自適應學習率，與 Adam 不同！

NumPy 矩陣運算 ── 科三 Q40
NumPy 核心操作 import numpy as np v1 = np.array([1, 2, 3]) v2 = np.array([4, 5, 6]) A = np.array([[1,2],[3,4]]) # ① 向量點積（Dot Product） np.dot(v1, v2) # = 1×4+2×5+3×6 = 4+10+18 = 32 → np.int64 v1 * v2 # = [4,10,18]（逐元素相乘，不是點積！） # ② 矩陣逆（Inverse） np.linalg.inv(A) # 計算 A 的反矩陣（不是行列式！） # ③ 特徵值分解（Eigendecomposition） np.linalg.eig(A) # 計算特徵值/特徵向量（不是反矩陣！） # ④ 行列式（Determinant） np.linalg.det(A) # 計算行列式值
函數功能容易混淆
np.dot(v1,v2)向量點積（純量）v1*v2 = 逐元素乘積（陣列）
np.linalg.inv(A)矩陣反矩陣常誤選為「行列式」
np.linalg.eig(A)特徵值/特徵向量常誤選為「反矩陣」
np.linalg.det(A)行列式值

📋 題目：X_train -= X_train.mean(axis=0) 和 X_train /= X_train.std(axis=0)，哪些敘述正確？
特徵標準化 # === 標準化（Z-score Normalization）=== # 讓每個特徵：均值=0、標準差=1 # ✅ A 正確：去均值 → 每個特徵的平均值調整為 0 X_train -= X_train.mean(axis=0) # 減去每個特徵的平均值 # ✅ D 正確：標準化後有助防止梯度爆炸/消失 X_train /= X_train.std(axis=0) # 除以標準差 → 標準差=1 # ❌ B 錯誤：std(axis=0) 讓標準差=1，不是=0 # ❌ C 錯誤：結果是均值0、std1，不是壓縮到[0,1] # → [0,1] 是 Min-Max 正規化！ # Min-Max（對比） X_train = (X_train - X_train.min()) / (X_train.max() - X_train.min()) # ↑ 這才會壓縮到 [0,1]
✅ 科三 Q48 答案 C（A+D正確）：
A：mean(axis=0) → 每特徵均值變 0 ✅
D：標準化可防止梯度爆炸/消失 ✅
B：std 變的是「= 1」不是「= 0」❌
C：壓縮到[0,1] 是 Min-Max，不是 Z-score ❌

⑤ Early Stopping 策略 科三 Q31
科三 Q31：驗證損失波動時的正確 Early Stopping 策略
選項是否正確原因
A：根據訓練集損失最低點停止❌ 錯訓練損失低 ≠ 泛化好；可能過擬合
B：監控驗證集損失 + 設定 Patience✅ 正確patience=N 代表「連續 N 輪無改善才停」，容忍波動
C：改以測試集作為早停依據❌ 嚴重錯誤測試集不能參與訓練決策，會造成 Data Leakage
D：合併所有資料訓練至收斂❌ 錯沒有驗證集就無法偵測過擬合
✅ 關鍵：Early Stopping 監控的是驗證集（validation）損失，而非訓練集損失。
Patience = 寬容度，防止因暫時波動提早停止。
⑥ 交叉驗證完整類型 科三 Q1 / Q34 / Q46
四種 CV 的選擇邏輯
方法適用場景優點缺點
K-Fold CV一般回歸/分類，資料量中等標準方法，效率高類別不均衡時可能偏差
Stratified K-Fold類別不均衡資料（推薦）確保每 fold 類別比例一致略增計算量
LOO-CV（Leave-One-Out）資料極少（<100筆）最大化訓練資料利用計算量大 O(n×訓練)
Stratified LOO-CV ⭐小樣本 + 類別不均衡（科三 Q34）最大利用率 + 類別平衡計算量最大
Time Series CV時序資料（不能隨機切）防止未來資料洩漏訓練集遞增，早期 fold 偏小
✅ Q34 情境：150 筆資料 + 8% 陽性（極不均衡） → 選 Stratified LOO-CV（分層留一法）
理由：樣本少 → LOO 最大化利用；類別不均衡 → Stratified 確保每次評估有正例
✅ Q1 情境：評估模型泛化能力 → 交叉驗證（Cross-Validation），不是 F-test 或 Chi-square test（那是假設檢定）
⑦ 偏差類型辨識 科三 Q13 / Q28
三大偏差類型一眼識別
Sampling Bias（取樣偏差）
訓練資料的樣本選取不具代表性，不能代表目標母體。

例：只用「活躍顧客」訓練，模型對新顧客預測差。（科三 Q28）
🔑 關鍵字：樣本選取、不代表母體
Label Bias（標籤偏差）
標記者的主觀偏見導致標籤不準確或不一致。

例：標記者文化背景不同，導致「正面/負面」情感標注標準不一。（科三 Q13）
🔑 關鍵字：主觀偏見、標記者
Confirmation Bias（確認偏差）
只收集或關注符合已有假設的資料，忽視矛盾證據。

例：只分析支持某結論的用戶回饋，忽略反對意見。
🔑 關鍵字：選擇性收集、假設確認
Feature Bias（特徵偏差）
使用了與目標高度相關但不應使用的特徵（如性別、種族），導致歧視性預測。

例：信用評分模型用郵遞區號作為代理種族特徵。
🔑 關鍵字：不公平特徵、代理變數
