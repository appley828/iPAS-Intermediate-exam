🔍 缺失值處理策略
缺失值處理方法比較：從簡單刪除到多重插補
缺失值類型（MCAR / MAR / MNAR）
MCAR（完全隨機缺失）：與任何變數無關，可直接刪除
MAR（隨機缺失）：與其他可觀測變數相關，用填補法
MNAR（非隨機缺失）：與缺失值本身相關，最難處理
▶ 考點：MNAR 最難處理
均值 vs 中位數填補選擇
資料對稱分佈→使用均值填補
資料偏態/有離群值→使用中位數填補（更穩健）
類別變數→使用眾數填補
填補後標準差會人為縮小
▶ 考點：偏態資料用中位數
KNN 填補原理
找 K 個最相似的完整樣本，取其平均值填補
優點：考慮特徵間相關性
缺點：計算成本高 O(n²)，大資料集慢
K 值選擇影響填補品質
▶ 考點：KNN 考慮特徵相關性
📌 iPAS 必記：缺失率 > 40% 考慮刪除整欄、中位數填補對偏態/離群值更穩健、KNN 填補計算成本 O(n²)、MNAR 缺失最難處理（需領域知識）
⚠️ 異常值偵測（IQR & Z-score）
IQR 法與 Z-score 法：核心公式與適用情境
// IQR（四分位距）異常值偵測
IQR = Q3 - Q1
下界 = Q1 - 1.5 × IQR
上界 = Q3 + 1.5 × IQR
超出範圍者 → 異常值

// Z-score 標準化異常值偵測
z = (x - μ) / σ
|z| > 3 → 異常值（±3σ 之外，約佔 0.3%）

// 修正 Z-score（對離群值更穩健）
MAD = median(|xᵢ - median(X)|)
修正 z = 0.6745 × (x - median) / MAD
IQR 法優勢
不假設資料分佈（非參數法）
對偏態分佈仍有效
箱形圖（Box Plot）直觀呈現
1.5×IQR 為標準閾值
3×IQR 更寬鬆（極端異常值）
▶ 考點：IQR 不假設常態分佈
Z-score 法限制
假設資料近似常態分佈
對非常態資料效果差
閾值 |z|>3 對應 99.7% 範圍
Masking 問題：多個異常值會影響均值/標準差計算
小樣本不適合使用
▶ 考點：Z-score 假設常態分佈
異常值處理策略
1. 刪除：確定是測量誤差
2. 截尾（Winsorizing）：替換為閾值
3. 對數轉換：壓縮偏態分佈
4. 保留：詐欺偵測中異常值本身是目標
5. Robust 縮放：降低異常值影響
▶ 考點：詐欺偵測保留異常值
📏 特徵縮放方法（互動比較）
三大縮放方法：Min-Max、Z-score 標準化、Robust Scaling
// Min-Max 正規化（MinMaxScaler）
x' = (x - x_min) / (x_max - x_min) → 範圍 [0, 1]

// Z-score 標準化（StandardScaler）
x' = (x - μ) / σ → 均值=0，標準差=1

// Robust Scaling（對離群值穩健）
x' = (x - median) / IQR → 使用中位數與四分位距
Min-Max 正規化
保留原始分佈形狀
輸出嚴格在 [0,1]
對神經網路激活函數友好
缺點：對離群值敏感（一個極端值壓縮其他值）
▶ 考點：神經網路/影像像素常用
Z-score 標準化
標準化後：均值=0、標準差=1
假設特徵近似常態分佈
SVM、PCA、線性迴歸適合
輸出無固定範圍（可超過±1）
▶ 考點：標準化後 σ=1, μ=0
Robust Scaling
使用中位數與 IQR 取代均值與標準差
對離群值最穩健
資料含大量異常值時優選
不保證輸出範圍
▶ 考點：含離群值→Robust Scaling
縮放方法公式輸出範圍對離群值適用情境
Min-Max(x-min)/(max-min)[0, 1]敏感神經網路、影像
Z-score 標準化(x-μ)/σ無固定範圍中等SVM、PCA、線性模型
Robust Scaling(x-median)/IQR無固定範圍最穩健含離群值資料集

🏷️ 類別特徵編碼方法
One-Hot / Label / Target / Frequency Encoding — 選擇策略與考試重點
// One-Hot Encoding（虛擬變數）
顏色 = ['紅','藍','綠']
紅 → [1, 0, 0], 藍 → [0, 1, 0], 綠 → [0, 0, 1]
// 注意：k 個類別 → k-1 個虛擬變數（避免完全共線性）

// Label Encoding（序號編碼）
低=0, 中=1, 高=2 → 適合有序類別（Ordinal）

// Target Encoding（目標編碼）
城市 = mean(target | 城市='台北') → 用目標均值替換類別

// Frequency Encoding（頻率編碼）
城市編碼 = count(城市) / total → 用出現頻率替換
One-Hot 編碼
無序類別首選（顏色、城市）
高基數問題：1000 個城市→1000 列，特徵爆炸
需刪除一列避免完全共線性（drop_first=True）
樹模型不需要 One-Hot（可直接處理類別）
▶ 考點：高基數→特徵爆炸
Target Encoding
用目標均值替換類別，保留類別-目標相關性
高基數特徵（城市、郵遞區號）適用
目標洩漏風險：需在 CV 每折內計算
加平滑：α/(α+count)×global_mean 防止過擬合
▶ 考點：Target Encoding 需防洩漏
Label vs Ordinal Encoding
Label Encoding：任意序號，引入人工序關係
誤用情境：無序類別用 Label → 模型誤認距離
Ordinal Encoding：有序類別必選（低/中/高）
樹模型：Label Encoding 可行（不假設線性關係）
▶ 考點：無序類別不能用 Label
📌 iPAS 必記：One-Hot 高基數→特徵爆炸（維度詛咒）、Label Encoding 給無序類別→引入假序關係、Target Encoding 需在 CV 每折內計算（防目標洩漏）
🔬 特徵選擇方法（Filter / Wrapper / Embedded）
三大特徵選擇流派：統計過濾、包裝法、嵌入法
方法類別代表算法原理計算成本過擬合風險
Filter（過濾法）卡方檢定、互信息、相關係數統計指標評分，獨立於模型低 O(p)低
Wrapper（包裝法）RFE、前向/後向選擇用模型評估特徵子集效能高 O(2ᵖ)高
Embedded（嵌入法）L1 Lasso、RandomForest 重要性訓練時內建特徵選擇中中
卡方檢定（SelectKBest）
適用：非負特徵（計數、頻率）與類別目標
H₀：特徵與目標相互獨立
p-value < 0.05 → 拒絕 H₀ → 特徵顯著
χ² 值越大 → 特徵與目標關聯越強
▶ 考點：卡方→非負特徵分類問題
Lasso L1 正則化特徵選擇
L1 懲罰項將不重要特徵係數壓縮為精確 0
自動特徵選擇（稀疏解）
λ（正則化強度）↑ → 更多特徵被置零
比 Filter 更考慮特徵間相互作用
▶ 考點：Lasso 產生稀疏解
Random Forest 特徵重要性
基於每個特徵在所有樹中的平均不純度降低量
（Gini 重要性 / Mean Decrease Impurity）
置換重要性（Permutation Importance）：打亂特徵後性能降幅
可處理非線性交互效果
▶ 考點：RF 重要性 = Gini 下降量
⚖️ 不平衡資料處理（SMOTE / Class Weight）
類別不平衡問題：詐欺偵測、疾病診斷常見挑戰
// SMOTE（合成少數類過採樣技術）
SMOTE 原理：
1. 對每個少數類樣本，找 K 個最近鄰
2. 隨機選一個鄰居，在連線上生成合成樣本
新樣本 = x + λ × (x_neighbor - x), λ ∈ [0,1]

// 類別不平衡評估指標（不用 Accuracy！）
Precision = TP / (TP + FP) → 預測正確率
Recall = TP / (TP + FN) → 真實召回率（不漏報）
F1 = 2 × P × R / (P + R) → 平衡精準與召回
AUC-ROC：面積越大 → 模型區分能力越強
SMOTE 過採樣
生成少數類合成樣本（非複製）
SMOTE-NC：混合型數值+類別特徵
SMOTE-Tomek：過採樣後再清理邊界
注意：SMOTE 只在訓練集使用，不能洩漏到測試集
▶ 考點：SMOTE=合成少數類樣本
Class Weight 類別權重
在損失函數中給少數類更大懲罰
sklearn: class_weight='balanced'
balanced: w_i = n_samples/(n_classes × n_i)
不增加資料量，計算成本低
適合樣本極度不平衡（1:1000）
▶ 考點：class_weight 不增加資料
欠採樣（Undersampling）
隨機刪除多數類樣本
TomekLinks：刪除兩類邊界樣本
NearMiss：選邊界附近的多數類
缺點：丟失多數類有用資訊
適合資料量充足且多數類資訊冗餘
▶ 考點：欠採樣丟失資訊
📌 iPAS 必記：不平衡資料→不能用 Accuracy（用 F1/AUC）、SMOTE 合成新樣本非複製、SMOTE 只用在訓練集、Class Weight 比 SMOTE 更快（無額外樣本）

pandas 資料清洗核心 API
# 缺失值處理
df.isnull().sum() # 統計每欄缺失數量
df.fillna(df.mean()) # 均值填補（數值欄）
df.fillna(method='ffill') # 前向填補（時序資料）
df.dropna(thresh=0.6*len(df)) # 刪除非空值<60%的欄位

# 類別編碼
pd.get_dummies(df['city'], drop_first=True) # One-Hot，刪除第一列
df['grade'] = df['grade'].map({'低':0,'中':1,'高':2}) # 序號編碼

# 異常值偵測（IQR）
Q1 = df.quantile(0.25); Q3 = df.quantile(0.75)
IQR = Q3 - Q1
mask = ~((df < Q1 - 1.5*IQR) | (df > Q3 + 1.5*IQR)).any(axis=1)
fillna 核心參數
fillna(value)：固定值填補
fillna(df.mean())：均值填補
fillna(df.median())：中位數填補
fillna(method='ffill')：前向填補（時序）
fillna(method='bfill')：後向填補
▶ 偏態用 median，時序用 ffill
get_dummies 注意點
drop_first=True：刪除第一個虛擬變數防共線
k 類別 → k 列（不設）或 k-1 列（設 drop_first）
dtype 參數控制輸出類型（bool/int）
columns 參數指定要編碼的欄位名
▶ 線性模型需 drop_first=True
quantile 分位數計算
df.quantile(0.25)：Q1 第一四分位數
df.quantile(0.75)：Q3 第三四分位數
df.quantile([0.25,0.5,0.75])：一次計算多個
IQR = Q3 - Q1
異常值：小於 Q1-1.5×IQR 或大於 Q3+1.5×IQR
▶ IQR 閾值 1.5 需記住

fit_transform vs transform
fit_transform(X_train)：學習統計量並轉換
transform(X_test)：用訓練統計量轉換
錯誤：在 X_test 上 fit_transform → 資料洩漏
scaler.mean_ 存儲訓練集均值
scaler.scale_ 存儲訓練集標準差
▶ 測試集只能用 transform
SelectKBest 參數
chi2：非負數值 + 類別目標（非負數！）
f_classif：ANOVA F 統計（連續特徵）
mutual_info_classif：互信息（非線性關係）
k：選擇前 k 個特徵
k='all'：計算分數但不過濾
▶ chi2 要求非負輸入
RobustScaler 適用情境
RobustScaler(quantile_range=(25,75))：預設
with_centering=True：減去中位數
with_scaling=True：除以 IQR
離群值高占比（>5%）時優於 StandardScaler
不保證輸出範圍在 [0,1]
▶ 離群值多→RobustScaler

Q1：以下 sklearn 程式碼中，哪行會導致資料洩漏？
scaler = StandardScaler()
(A) X_train_scaled = scaler.fit_transform(X_train)
✓ (B) X_test_scaled = scaler.fit_transform(X_test)
(C) X_test_scaled = scaler.transform(X_test)
(D) pipe = Pipeline([('scaler', scaler), ('clf', clf)])
✓ 解析：測試集不能 fit_transform，應只 transform（使用訓練集統計量），否則測試集的均值/標準差資訊流入模型評估，導致過於樂觀的效果
Q2：使用 SMOTE 時，以下哪個操作是正確的？
(A) sm.fit_resample(X, y)（整個資料集）
(B) sm.fit_resample(X_test, y_test)（測試集）
✓ (C) sm.fit_resample(X_train, y_train)（僅訓練集）
(D) 在 train_test_split 之前執行 SMOTE
✓ 解析：SMOTE 必須只在訓練集上執行，若在完整資料集或測試集上執行，合成樣本可能出現在測試集，導致評估失真（資料洩漏）
