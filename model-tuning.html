⚖️ Bias-Variance Tradeoff（偏差-方差權衡）
欠擬合 vs 過擬合：偏差與方差的拉鋸
// 模型泛化誤差分解（EPE）
Error(x) = Bias² + Variance + Irreducible Noise

// Bias（偏差）：模型假設過於簡單，無法捕捉資料規律
Bias = E[ŷ] - y
// Variance（方差）：模型對訓練資料過度敏感，泛化差
Variance = E[(ŷ - E[ŷ])²]

// 模型複雜度 ↑ → Bias ↓、Variance ↑（Tradeoff 存在）
Sweet Spot = 最小化 Bias² + Variance 的模型複雜度
欠擬合（High Bias）
訓練誤差高、測試誤差高
模型太簡單（如線性模型擬合非線性資料）
解法：增加模型複雜度、增加特徵、減少正則化強度
診斷：學習曲線中訓練分數持續偏低
▶ 考點：High Bias = 欠擬合 = 訓練誤差高
過擬合（High Variance）
訓練誤差低、測試誤差高（兩者差距大）
模型記憶訓練資料噪聲
解法：增加資料、正則化、Dropout、減少特徵
診斷：學習曲線中訓練與驗證分數差距大
▶ 考點：High Variance = 過擬合 = 訓練vs測試誤差差距大
診斷策略
學習曲線（Learning Curve）：隨訓練樣本增加觀察誤差變化
驗證曲線（Validation Curve）：隨超參數變化觀察誤差
兩者都高 → 欠擬合；差距大 → 過擬合
▶ 考點：學習曲線診斷欠/過擬合
互動圖：拖動滑桿調整模型複雜度，觀察偏差²與方差的變化
● 偏差²（Bias²） ● 方差（Variance） ● 總誤差（Error）
📌 iPAS 必記：欠擬合→High Bias→訓練誤差也高；過擬合→High Variance→訓練誤差低但測試誤差高；最佳點在 Bias²+Variance 最小處
🔒 正則化（Regularization）
L1（Lasso）、L2（Ridge）、Elastic Net 與 Dropout 比較
// L1 Lasso：損失函數加絕對值懲罰
Loss_L1 = MSE + λ × Σ|wᵢ|
效果：部分權重變 0（稀疏解），自動特徵選擇

// L2 Ridge：損失函數加平方懲罰
Loss_L2 = MSE + λ × Σwᵢ²
效果：所有權重均勻縮小，不會變 0，適合多重共線性

// Elastic Net：L1 + L2 線性組合
Loss_EN = MSE + λ₁Σ|wᵢ| + λ₂Σwᵢ²
效果：兼具稀疏性與穩定性，適合高維稀疏資料
L1 vs L2 幾何直覺
L1：約束區域為菱形，最優解常落在頂點（稀疏）
L2：約束區域為圓形，最優解均勻分布在圓上
λ 越大 → 懲罰越強 → 欠擬合風險增加
▶ 考點：L1 稀疏=特徵選擇；L2 均勻縮小
Dropout 正則化
訓練時隨機關閉比例 p 的神經元
測試時使用全部神經元，輸出乘以 (1-p) 或訓練時縮放
效果：防止神經元之間的共同適應（co-adaptation）
p = 0.5 常見於全連接層
▶ 考點：Dropout 防過擬合，測試時不 Dropout
正則化強度 λ 選擇
λ 過小：正則化效果弱，仍過擬合
λ 過大：模型過於簡單，欠擬合
最佳 λ 通過交叉驗證搜索
Ridge regression 有解析解，計算效率高
▶ 考點：λ 需交叉驗證調整
📌 iPAS 必記：L1 產生稀疏解（部分係數=0）可做特徵選擇；L2 係數均勻縮小不會為零；Dropout 隨機關閉神經元防過擬合；Elastic Net = L1+L2
🔁 交叉驗證（Cross-Validation）
k-fold、Stratified k-fold、Leave-One-Out：選擇策略與計算成本
方法原理計算成本適用情境考點
k-fold CV 資料分 k 折，輪流用 1 折驗證、k-1 折訓練 k 倍訓練成本 一般情境，k=5 或 k=10 最常用 結果取 k 次平均，更穩定
Stratified k-fold 每折保持各類別比例與原始分布一致 k 倍訓練成本 分類問題、資料不平衡 防止某折缺少少數類樣本
Leave-One-Out（LOO） 每次只用 1 個樣本驗證，其餘訓練（k=n） n 倍訓練成本（最高） 樣本量極少（n<50） 方差高、計算成本最大
Time Series CV 保持時間順序，訓練集不能包含未來資料 k 倍訓練成本 時間序列預測 禁止隨機 shuffle（避免資料洩漏）
// k-fold CV 評估流程
for i in range(k):
  val_fold = data[ith fold]
  train_fold = data[remaining k-1 folds]
  model.fit(train_fold)
  scores[i] = model.score(val_fold)
cv_score = mean(scores) ± std(scores) // ±std 反映穩定性
k 值選擇策略
k=5：計算成本低、方差稍高
k=10：最常用黃金標準、偏差與方差均衡
k=n（LOO）：無偏估計但計算昂貴、方差極大
資料越少 → 建議用更大的 k 值
▶ 考點：k=5 or 10 最常用
資料洩漏防範
Scaler/PCA 等前處理必須在每折訓練集上 fit
不能在整個資料集上 fit 再 transform（資料洩漏）
使用 sklearn Pipeline 自動防止洩漏
測試集一律不能在驗證前接觸
▶ 考點：前處理必須只用訓練集 fit
驗證集 vs 測試集
驗證集：用於超參數調校（參與模型選擇）
測試集：最終評估，只用一次（不參與任何調校）
Nested CV：外層評估模型、內層選超參數
避免「測試集污染」導致性能高估
▶ 考點：測試集只用一次
📌 iPAS 必記：k-fold 計算成本為 k 倍；Stratified k-fold 保持類別比例，適合不平衡資料；LOO 計算成本最高（n 倍）；前處理 fit 必須在訓練集做
🔧 超參數調校（Hyperparameter Tuning）
Grid Search、Random Search 與 Bayesian Optimization 比較
Grid Search 窮舉搜索
對所有參數組合逐一評估
組合數 = Π（各參數候選值數量）
優點：找到最優解（若在搜索空間內）
缺點：維度詛咒，高維度時計算爆炸
▶ 考點：計算成本 = 組合數 × k-fold 次數
Random Search 隨機搜索
從參數空間隨機採樣 n 個組合評估
Bergstra 2012：高維空間中隨機搜索往往優於 Grid
原因：每個參數不一定都重要，隨機覆蓋更廣
計算成本可控，適合初步探索
▶ 考點：高維空間 Random Search 常優於 Grid
Bayesian Optimization
建立代理模型（如高斯過程）模擬目標函數
利用 Acquisition Function（期望改進）決定下一個候選點
優點：採樣效率高，適合昂貴的評估
工具：Optuna、Hyperopt、Ray Tune
▶ 考點：採樣效率最高，適合深度學習調參
📌 iPAS 必記：Grid Search 穷舉但計算成本高；Random Search 高維空間更有效；Bayesian Optimization 採樣效率最高；Early Stopping 監測驗證損失自動停止訓練
🧩 集成學習（Ensemble Learning）— 3D 投票視覺化
Bagging、Boosting、Stacking — 多模型集成投票動態展示
集成學習投票模型（拖動旋轉）
Bagging（Bootstrap Aggregating）
從訓練集有放回抽樣（Bootstrap）訓練多個基模型
預測結果取多數投票（分類）或平均（迴歸）
代表：隨機森林（額外加入特徵隨機選擇）
減少 Variance，對過擬合有效
▶ 考點：Bagging 減少方差，Random Forest 額外隨機特徵
Boosting（提升法）
順序訓練弱學習器，每個新模型聚焦前一個模型的錯誤樣本
AdaBoost：調整樣本權重
Gradient Boosting：擬合殘差梯度
XGBoost：加入正則化 + 二階梯度 + 並行計算，最強
▶ 考點：Boosting 減少偏差；XGBoost=梯度提升+正則化
Stacking（堆疊泛化）
第一層：多個不同類型的基模型（meta-learner 輸入）
第二層：元學習器（Meta-Learner）整合第一層預測
優點：充分利用不同模型的多樣性
需用 out-of-fold 預測避免資料洩漏
▶ 考點：Stacking 需防止資料洩漏；out-of-fold 是關鍵
🚀 Boosting 演進：AdaBoost → GBM → XGBoost
Boosting 家族演進與 XGBoost 核心技術
演算法核心思想特點考試重點
AdaBoost 調整樣本權重，讓新樹聚焦錯誤樣本 對噪聲敏感，容易過擬合 弱學習器組合為強學習器，樣本加權
Gradient Boosting（GBM） 擬合前一個模型的殘差梯度 計算慢（串行），但性能強 每棵樹擬合殘差，學習率控制步長
XGBoost GBM + L1/L2 正則化 + 二階梯度 + 並行 速度快、精度高、Kaggle 神器 正則化防過擬合、Column Subsampling
LightGBM Leaf-wise 生長 + 直方圖特徵分桶 速度比 XGBoost 快，適合大資料 Leaf-wise vs Level-wise 生長策略
// XGBoost 目標函數（帶正則項）
Obj = Σ L(yᵢ, ŷᵢ) + Σ Ω(fₖ)
Ω(f) = γT + ½λΣwⱼ² // T=葉節點數，w=葉節點值，γ=剪枝閾值

// 二階 Taylor 展開近似損失函數
Obj ≈ Σ [gᵢf(xᵢ) + ½hᵢf²(xᵢ)] + Ω(f)
gᵢ = ∂L/∂ŷᵢ // 一階梯度
hᵢ = ∂²L/∂ŷᵢ² // 二階梯度（Hessian）
XGBoost 關鍵超參數
n_estimators：樹的數量（越多越慢）
learning_rate (eta)：每棵樹的貢獻縮放（0.01~0.3）
max_depth：樹深（控制複雜度）
subsample：行採樣比例（0.6~0.9）
colsample_bytree：列採樣比例
▶ 考點：learning_rate + n_estimators 相互制衡
Random Forest vs XGBoost
Random Forest（Bagging）：並行訓練、對噪聲穩健
XGBoost（Boosting）：串行訓練、精度通常更高
RF 訓練快（可並行）；XGB 需 Early Stopping
XGB 對超參數更敏感，需仔細調校
▶ 考點：RF 並行訓練；XGB 串行（但內部計算並行）
集成多樣性原則
集成效果好的條件：
①基模型準確率 > 50%（優於隨機）
②基模型之間的錯誤相互獨立（多樣性）
多樣性來源：不同算法、不同特徵子集、不同訓練資料
▶ 考點：基模型需獨立且準確率>50%
📌 iPAS 必記：XGBoost = 梯度提升 + L1/L2 正則化 + 二階梯度 + 並行；Bagging 減少方差；Boosting 減少偏差；集成需基模型多樣性
📊 模型評估指標（互動比較）
分類 vs 迴歸評估指標核心考點
精確率 Precision vs 召回率 Recall
Precision = TP / (TP + FP)：預測為正的有多少真的是正
Recall = TP / (TP + FN)：所有真正例有多少被找到
F1 = 2×(P×R)/(P+R)：調和平均
Precision-Recall Tradeoff：閾值升高→P↑ R↓
▶ 考點：F1 = 2PR/(P+R)，P-R 有 Tradeoff
AUC-ROC 曲線
ROC：以 TPR vs FPR 繪圖（不同閾值）
AUC：ROC 曲線下面積，越接近 1 越好
AUC=0.5 相當於隨機猜測
優點：不受不平衡資料影響
▶ 考點：AUC=1 完美；=0.5 隨機猜
迴歸評估指標
MSE = Σ(y-ŷ)²/n：對離群值敏感
MAE = Σ|y-ŷ|/n：對離群值穩健
RMSE = √MSE：與目標同量綱
R²：解釋方差比例，1=完美，0=只預測均值
▶ 考點：MSE 懲罰大誤差；MAE 穩健；R² 越接近 1 越好
📌 iPAS 必記：不平衡資料用 F1/AUC 而非 Accuracy；Precision 關注誤報率；Recall 關注漏報率；R² 可為負（模型比均值還差）

🌲 正則化模型：Ridge、Lasso、RandomForest
# L1 Lasso 正則化迴歸
from sklearn.linear_model import Lasso, Ridge, ElasticNet
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline

pipe = Pipeline([
  ('scaler', StandardScaler()), # 正則化前必須標準化
  ('lasso', Lasso(alpha=0.01)) # alpha=λ 正則化強度
])
pipe.fit(X_train, y_train)
print("非零係數：", (pipe['lasso'].coef_ != 0).sum()) # L1稀疏特性

# Random Forest 特徵重要性
rf = RandomForestClassifier(n_estimators=200, max_features='sqrt')
rf.fit(X_train, y_train)
importances = rf.feature_importances_ # Gini impurity 降低量
Lasso alpha 選擇
alpha 越大 → 越多係數變 0 → 更稀疏
用 LassoCV 自動選擇最佳 alpha（內建 CV）
Lasso 不能處理高度相關特徵（會隨機選其一）
▶ LassoCV 自動調 alpha
Pipeline 防洩漏
Pipeline 中的 fit 只在訓練集執行
cross_val_score(pipe, X, y) 自動在每折正確 fit
避免在整個資料集 fit Scaler 後再 CV（資料洩漏）
▶ 考點：Pipeline 是防洩漏的標準做法
max_features 參數
'sqrt'：每棵樹隨機選 √n 個特徵（分類預設）
'log2'：每棵樹隨機選 log₂n 個特徵
None：使用全部特徵（退化為普通Bagging）
特徵隨機性是 RF 降低相關性的核心
▶ 考點：RF 特徵隨機選 √n

# XGBoost 訓練與 Early Stopping
import xgboost as xgb

)
model.fit(X_train, y_train,

  verbose=False)

Early Stopping
早停機制：驗證指標連續 n 輪無改善時停止
防止過擬合，自動決定最佳樹的數量
best_iteration_ 屬性記錄最佳輪數
▶ 考點：Early Stopping 防止過擬合
XGBoost 正則化參數
reg_alpha（α）= L1 正則化（稀疏葉節點值）
reg_lambda（λ）= L2 正則化（均勻縮小葉節點值）
gamma（min_split_loss）= 最小分割增益，相當於剪枝
▶ 考點：reg_alpha=L1，reg_lambda=L2
特徵重要性
importance_type='weight'：特徵被用來分裂的次數
importance_type='gain'：特徵帶來的平均增益（更準）
importance_type='cover'：特徵覆蓋的樣本數
xgb.plot_importance(model) 直接繪圖
▶ 考點：gain 反映特徵貢獻度最準確

程式題 1：下列 sklearn 程式碼有何問題？
scaler.fit(X) ; X_scaled = scaler.transform(X) ; cross_val_score(model, X_scaled, y, cv=5)
A. cross_val_score 的 cv 應設為 10 而不是 5
B. ✓ scaler 在整個 X 上 fit，導致驗證集資訊洩漏到訓練集（應用 Pipeline）
C. cross_val_score 無法與 scaler 一起使用
D. StandardScaler 不能用於分類問題
解析：在完整 X 上 fit Scaler，驗證集的統計量（均值/標準差）已洩漏進 Scaler，等於驗證集間接「看過」訓練集。正確做法是用 Pipeline 確保 Scaler 只在訓練折 fit。
程式題 2：XGBoost 中 early_stopping_rounds=50 的作用是？
A. 訓練 50 棵樹後強制停止，不管驗證集表現
B. 每 50 輪儲存一次模型 checkpoint
C. ✓ 驗證集指標連續 50 輪未改善則停止，自動防止過擬合
D. 僅適用於 n_estimators>500 時
