🔒 差分隱私（Differential Privacy）
ε-差分隱私：加入噪聲保護個別記錄，同時保留統計規律
// ε-差分隱私正式定義（鄰近資料集 D 與 D'）
Pr[M(D) ∈ S] ≤ e^ε × Pr[M(D') ∈ S]

// ε（隱私預算）越小 → 隱私保護越強，但資料效用越低
ε = 0：完美隱私（輸出與輸入完全無關，無效）
ε → ∞：無隱私保護
實務常用 ε = 0.1 ~ 10 的範圍

// 拉普拉斯噪聲機制（數值查詢）
M_Lap(f, D) = f(D) + Lap(Δf / ε)
// Δf = 全局敏感度（sensitivity），即改變一筆記錄對 f 的最大影響

// 高斯噪聲機制（(ε,δ)-差分隱私）
M_Gauss(f, D) = f(D) + N(0, σ²)
σ ≥ Δf × √(2ln(1.25/δ)) / ε
差分隱私核心概念
保護：即使攻擊者知道除某一筆之外的所有資料，也無法判斷該筆是否在資料集中
加入噪聲 → 輸出分布幾乎相同
ε 越小 → 噪聲越大 → 隱私更強但精度下降
▶ 考點：ε 控制隱私強度，越小越安全
拉普拉斯 vs 高斯
拉普拉斯：純 ε-DP，σ = Δf/ε，尖峰分布
高斯：(ε,δ)-DP，δ 為微小的隱私洩漏機率，鐘型分布
高斯噪聲更適合深度學習（DP-SGD）
▶ 考點：Lap 用於計數查詢；Gaussian 用於 DP-SGD
全局敏感度（Δf）
定義：改變資料集中一筆記錄，查詢結果 f 的最大變化量
計數查詢（COUNT）：Δf = 1
求和查詢（SUM）：Δf = max value
均值查詢（MEAN）：Δf = (max-min)/n
▶ 考點：敏感度決定需加多大噪聲
📌 iPAS 必記：差分隱私加噪聲保護個別記錄；ε 越小隱私保護越強但資料效用越低；拉普拉斯噪聲用於純 ε-DP；高斯噪聲用於 (ε,δ)-DP；Apple、Google 均有部署本地差分隱私
🎭 資料匿名化技術
k-匿名、ℓ-多樣性、t-接近性：保護隱私的三道防線
技術定義防禦攻擊弱點考點
k-匿名（k-Anonymity） 每筆記錄與至少 k-1 筆其他記錄在準識別符上無法區分 連結攻擊（Linkage Attack） 同質性攻擊：若 k 個人敏感屬性相同，即可推測 k=2 最少需 2 筆相同；k 越大越安全
ℓ-多樣性（ℓ-Diversity） 每個等價類中敏感屬性至少有 ℓ 種不同值 同質性攻擊 偏斜攻擊：若 ℓ 個值中 1 個佔 99%，仍可推測 補強 k-匿名的敏感屬性多樣性
t-接近性（t-Closeness） 等價類中敏感屬性分布與整體資料集分布差距 ≤ t 偏斜攻擊、背景知識攻擊 資料可用性下降最多 最強保護，t 越小越嚴格
資料概化（Generalization） 將精確值替換為範圍（如年齡 32→30-40） 直接識別攻擊 細粒度資訊損失 實作 k-匿名的主要手段
隨機擾動（Perturbation） 對數值加入隨機噪聲或交換值 精確推斷攻擊 統計性質改變 差分隱私的實作基礎
// k-匿名範例：年齡與郵遞區號概化
原始: 年齡=32, 郵遞區號=10045, 疾病=癌症
k-匿名(k=3): 年齡=[30-35], 郵遞區號=100**, 疾病=癌症

// ℓ-多樣性：同一等價類的敏感屬性種類 ≥ ℓ
等價類 {30-35, 100**}: 疾病 = {癌症, 糖尿病, 心臟病} → ℓ=3

// 準識別符（Quasi-Identifier）：間接識別個人的屬性組合
例：年齡 + 性別 + 郵遞區號 → 可能唯一識別個人
準識別符（QI）概念
單一屬性不能識別個人，但組合後可以
例：87% 美國人可由{出生日期, 性別, 5碼郵遞區號}唯一識別
HIPAA Safe Harbor 方法：移除18種識別符
▶ 考點：QI 組合才導致隱私洩漏
k-匿名實作流程
1. 找出準識別符（QI）
2. 排序資料
3. 概化 QI 使每個等價類 ≥ k 筆
4. 驗證敏感屬性多樣性（ℓ-Diversity）
Mondrian 演算法：貪婪切分多維 QI 空間
▶ 考點：k-匿名 = QI 概化 + 等價類大小
匿名化 vs 去識別化
匿名化（Anonymization）：無法再次識別，GDPR 視為非個資
去識別化（De-identification）：移除直接識別符，仍可能透過 QI 再識別
假名化（Pseudonymization）：替換識別符，持有映射表可還原，仍屬個資
▶ 考點：真正匿名才脫離 GDPR 管轄
📌 iPAS 必記：k-匿名每筆記錄至少與 k-1 筆相同；ℓ-多樣性補強敏感屬性種類；k-匿名無法防同質性攻擊；差分隱私是比 k-匿名更強的數學保證

⚖️ 公平性指標（Fairness Metrics）
Demographic Parity、Equal Opportunity、Disparate Impact 80% 原則
// 1. 人口統計均等（Demographic Parity）
Pr[Ŷ=1 | A=0] = Pr[Ŷ=1 | A=1]
// 不同保護群體 A 應有相同的正面預測率

// 2. 機會均等（Equal Opportunity）
Pr[Ŷ=1 | Y=1, A=0] = Pr[Ŷ=1 | Y=1, A=1]
// 條件：實際為正（Y=1）的情況下，不同群體的 True Positive Rate 相等

// 3. 差異影響比（Disparate Impact Ratio）
DI = Pr[Ŷ=1 | A=disadvantaged] / Pr[Ŷ=1 | A=advantaged]
// EEOC 80% 原則：DI < 0.8 → 構成間接歧視（Adverse Impact）
例：A群體通過率 40%，B群體通過率 60% → DI = 40/60 = 0.67 < 0.8 ⚠️

// 4. 校準公平性（Calibration）
Pr[Y=1 | score=s, A=0] = Pr[Y=1 | score=s, A=1]
// 相同分數代表相同的實際風險，不因群體不同而異
公平性指標定義關注重點適用場景不可能定理
Demographic Parity 各群體正面預測率相等 決策分配均等 招聘、貸款審批 與校準公平性不相容
Equal Opportunity 真正率（TPR）跨群體相等 真正合格者不被歧視 刑事司法、醫療篩查 與 Predictive Parity 不相容
Equalized Odds TPR 與 FPR 同時跨群體相等 錯誤率均等 需平衡誤判成本的場景 更難同時滿足
Disparate Impact 弱勢群體通過率 ÷ 優勢群體通過率 結果比例差異 美國 EEOC 法規標準 <0.8 構成間接歧視
Disparate Impact 80% 原則計算
步驟：
1. 計算各群體的正面決策率
2. 以最低率除以最高率
3. 結果 < 0.8（80%）→ 間接歧視
例：男性錄用率 50%，女性 35%
DI = 35/50 = 0.70 < 0.8 → 違反
▶ 考點：DI < 0.8 = 間接歧視（Adverse Impact）
公平性不可能定理
Chouldechova (2017) 等証明：
在不完美分類器且群體基本率不同時，Demographic Parity、Equal Opportunity、Calibration 無法同時滿足
→ 必須根據業務場景選擇最重要的公平性定義
▶ 考點：公平性指標之間存在根本衝突

ProPublica (2016)：COMPAS 刑事再犯風險評估

🔍 模型可解釋性（Explainability）
SHAP、LIME、PDP：黑盒模型的透明化工具
// SHAP（SHapley Additive exPlanations）：博弈論 Shapley 值
ϕᵢ(f) = Σ [|S|!(n-|S|-1)!/n!] × [f(S∪{i}) - f(S)]
// ϕᵢ：特徵 i 的 SHAP 值 = 該特徵對預測貢獻的平均邊際效果
輸出 = 基準值 + Σ ϕᵢ（所有 SHAP 值之和 = 預測 - 基準預測）

// LIME（Local Interpretable Model-agnostic Explanations）
ξ(x) = argmin [L(f, g, π) + Ω(g)]
// 在樣本 x 附近建立可解釋的線性代理模型 g
// L：複雜模型 f 與簡單模型 g 在 x 附近的差距；Ω：g 的複雜度懲罰
方法原理範圍優點缺點考點
SHAP Shapley 值（博弈論）計算特徵貢獻 全局 + 局部 一致性、準確性；有理論保證 計算複雜度高（2^n） 特徵重要性排名最可靠
LIME 在預測附近建立局部線性代理 局部 模型無關（任何黑盒）；易理解 採樣隨機性→結果不穩定 解釋單一預測的首選
PDP（部分依賴圖） 邊際化其他特徵，展示單特徵平均效果 全局 視覺化特徵-輸出關係 忽略特徵交互效應 展示特徵的整體趨勢
ICE 圖（個體條件期望） PDP 的逐樣本版，展示異質性效果 局部集合 揭示個體差異（PDP 的延伸） 圖形複雜難解讀 發現交互效應的補充
Attention Weights Transformer 注意力權重視覺化 局部 原生於 Transformer 架構 不一定代表因果關係 NLP/CV 模型解釋入門
SHAP 值解讀
正 SHAP 值：該特徵推動預測值升高
負 SHAP 值：該特徵推動預測值降低
|SHAP| 越大 → 特徵對此預測影響越大
Waterfall Plot：單一樣本逐特徵貢獻疊加
Beeswarm Plot：全資料集特徵重要性概覽
▶ 考點：SHAP 解釋特徵重要性的業界標準
可解釋性 vs 準確性
線性模型/決策樹：本質可解釋（Intrinsic）
黑盒模型（DNN/XGBoost）：事後解釋（Post-hoc）
EU AI Act 要求高風險 AI 必須可解釋
GDPR 第22條：自動化決策需提供解釋
▶ 考點：高風險 AI 法規要求可解釋性
TreeExplainer vs KernelExplainer
TreeExplainer：專為樹模型設計（XGBoost/RF），O(TLD²) 精確計算，速度快
KernelExplainer：模型無關，透過 LIME 思路近似 Shapley 值
LinearExplainer：線性模型解析解，最快
DeepExplainer：深度學習，基於 DeepLIFT
▶ 考點：TreeExplainer 最適合 XGBoost
📌 iPAS 必記：SHAP 基於 Shapley 值，解釋特徵貢獻最可靠；LIME 建立局部線性代理，解釋單一預測；PDP 展示特徵平均效果；GDPR 第22條賦予自動化決策解釋權
📜 GDPR 與台灣個資法
個人資料保護法規：適用範圍、主要義務與違規罰則
面向EU GDPR（2018）台灣個資法（2023修正）
適用範圍 處理 EU 居民個人資料的所有組織（含境外） 台灣境內蒐集、處理、利用個人資料之公私機構
個資定義 可識別自然人的任何資訊（含 IP、Cookie、位置） 足以識別個人之姓名、出生日期、身分證字號等資料
六大合法基礎 同意/契約/法律義務/重要利益/公共任務/正當利益 當事人同意/法律明文/公共利益/當事人個人利益
資料主體權利 知情權、存取權、更正權、刪除權（被遺忘權）、可攜權、反對權 查詢、閱覽、複製、補充更正、請求停止蒐集/刪除
資料外洩通知 72小時內通知主管機關；高風險需通知當事人 查知後在合理時間內通知主管機關及當事人
最高罰款 全球年營收 4% 或 2000萬歐元（取較高者） 非公務機關：最高罰款新台幣 1500萬元
跨境傳輸 需確保第三國有充分保護（Adequacy Decision） 中央主管機關限制傳輸目的地
GDPR 對 AI 的特別規範
第22條：禁止完全依賴自動化決策（含剖析）做出重大決定
隱私設計（Privacy by Design）：系統設計時就納入隱私保護
資料最小化：只蒐集必要資料
目的限制：資料只能用於蒐集時的目的
DPO（資料保護官）：大規模處理需指派
▶ 考點：GDPR 第22條=自動化決策解釋權

🌐 AI 倫理框架（AI Ethics Framework）
透明度、問責性、公平性、無害性：負責任 AI 的四大支柱
倫理原則定義實踐方式相關指標/工具
透明度（Transparency） AI 系統的運作方式與決策過程可被理解 Model Card、可解釋 AI（XAI）、開源程式碼 SHAP、LIME、決策樹代理
問責性（Accountability） AI 決策的責任歸屬明確，有問責機制 稽核日誌、人工審查、責任鏈設計 MLflow 追蹤、版本控制
公平性（Fairness） AI 決策不對特定群體造成歧視或不公平結果 偏見審計、公平性約束、多元資料蒐集 Demographic Parity、DI Ratio
無害性（Non-maleficence） AI 系統不對個人或社會造成傷害 風險評估、對抗測試（Red Teaming） 風險矩陣、AI 安全評估
隱私（Privacy） 保護個人資料，尊重資料主體權利 差分隱私、資料最小化、匿名化 GDPR 合規工具、隱私影響評估（PIA）
人類自主性（Human Autonomy） AI 輔助而非取代人類決策；保留人工覆核 Human-in-the-Loop、反對自動化決策的權利 GDPR 第22條、EU AI Act 人工監督

台灣 AI 倫理指引：呼應 OECD，加入科技主權概念
▶ 考點：各框架核心原則高度重疊

# scipy.stats 統計假設檢定
from scipy import stats

# 卡方獨立性檢定：測試模型決策是否與敏感屬性獨立
contingency = [[90, 10], [60, 40]] # [[群體A通過/拒絕],[群體B通過/拒絕]]
chi2, p, dof, expected = stats.chi2_contingency(contingency)
# p < 0.05 → 拒絕獨立性假設 → 決策與群體相關 → 可能存在偏見

# 雙樣本 t 檢定：比較兩群體分數是否有顯著差異
t_stat, p_val = stats.ttest_ind(group_a_scores, group_b_scores)

# 單因子 ANOVA：多個群體均值比較（超過 2 群體）
f_stat, p_val = stats.f_oneway(g1, g2, g3) # H₀: 所有群體均值相等
chi2_contingency
輸入：列聯表（觀察次數矩陣）
輸出：chi2統計量、p值、自由度、期望次數
用途：測試兩類別變數是否獨立（如決策 vs 群體屬性）
p < 0.05 → 兩變數不獨立（可能存在系統性差異）
▶ 考點：卡方檢定測試分類決策公平性
ttest_ind
用途：比較兩獨立樣本均值是否顯著不同
equal_var=False → Welch's t-test（不假設方差相等）
alternative='two-sided'（預設）/ 'less' / 'greater'
配合 cohen's d 衡量效應量（Effect Size）
▶ 考點：t-test 用於連續分數的群體比較

差分隱私實作：NumPy 噪聲添加
# 拉普拉斯機制：計數查詢差分隱私
import numpy as np

def laplace_mechanism(true_value, sensitivity, epsilon):
  # b = sensitivity / epsilon（拉普拉斯分布的尺度參數）
  b = sensitivity / epsilon
  noise = np.random.laplace(0, b)
  return true_value + noise

# 高斯機制：(ε, δ)-差分隱私
def gaussian_mechanism(true_value, sensitivity, epsilon, delta=1e-5):
  sigma = sensitivity * np.sqrt(2 * np.log(1.25 / delta)) / epsilon
  return true_value + np.random.normal(0, sigma)

# 使用範例：計數查詢（Δf=1）
true_count = 1500
private_count = laplace_mechanism(true_count, sensitivity=1, epsilon=0.5)
隱私預算 ε 選擇
ε = 0.1：強隱私，噪聲極大（Apple 設備統計）
ε = 1.0：中等隱私，常見研究設定
ε = 10：弱隱私，適合低敏感查詢
組合定理：多個查詢的 ε 會累加（隱私預算消耗）
▶ 考點：ε 越小越安全，但效用越低
np.random.laplace / normal
laplace(loc, scale)：loc=均值，scale=b=Δf/ε
normal(mean, std)：mean=0，std=σ
噪聲量：ε↓ → b↑ → 更大噪聲
計數查詢敏感度 Δf=1（增刪一筆記錄最多改變計數1）
▶ 考點：計數查詢 sensitivity=1
DP-SGD（差分隱私梯度下降）
訓練深度學習時每步梯度加噪聲：
1. 裁剪梯度（梯度裁剪上限 C）
2. 加高斯噪聲 N(0, σ²C²)
工具：TensorFlow Privacy、Opacus（PyTorch）
Accountant 追蹤累計隱私損耗

# 計算 Disparate Impact Ratio
import pandas as pd
from sklearn.metrics import confusion_matrix

def disparate_impact(y_pred, sensitive_attr):
  groups = pd.Series(sensitive_attr).unique()
  rates = {};
  for g in groups:
    mask = (sensitive_attr == g)
    rates[g] = y_pred[mask].mean() # 正面預測率
  return min(rates.values()) / max(rates.values())

di = disparate_impact(y_pred, sensitive_attr)
print(f"DI = {di:.3f}, {'✅ 通過' if di >= 0.8 else '❌ 可能歧視'}")
